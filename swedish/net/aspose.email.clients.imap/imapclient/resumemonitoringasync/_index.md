## ImapClient.ResumeMonitoringAsync method

Återupptar övervakning av meddelandeändringar för angiven mapp. Till skillnad från StartMonitoring-metoden kommer den att hitta alla saknade brevlådeändringar och ringa återuppringningen för dem.

```csharp
public Task ResumeMonitoringAsync(ImapMonitoringEventHandler callback, 
    ImapMonitoringErrorEventHandler errorCallback, IImapMonitoringState monitoringState, 
    CancellationToken token = default)
```

| Parameter | Typ | Beskrivning |
| --- | --- | --- |
| callback | ImapMonitoringEventHandler | Återuppringningsfunktionen för att övervaka driften. |
| errorCallback | ImapMonitoringErrorEventHandler | Återuppringningsfunktionen för att övervaka felhantering. Övervakning av angiven mapp stoppas när denna återuppringning anropas. Återuppringningen tillhandahåller också en tillståndsinnehavare så mappövervakning kan återupptas med`ResumeMonitoringAsync` metod. |
| monitoringState | IImapMonitoringState | Övervakningsläget att återuppta mappövervakning från. Kan hämtas från[`MonitoringState`](../../imapmonitoringerroreventargs/monitoringstate) . |
| token | CancellationToken | Avbokningstoken |
