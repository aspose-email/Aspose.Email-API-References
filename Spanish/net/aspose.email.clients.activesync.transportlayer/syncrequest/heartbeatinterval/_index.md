## SyncRequest.HeartbeatInterval property

Especifica la cantidad de segundos que el servidor DEBERÍA retrasar una respuesta si no se agregan elementos nuevos a las carpetas incluidas. Los valores válidos para HeartbeatInterval son de 60 a 3540 segundos (59 minutos). Cuando el cliente solicita un intervalo que está fuera del rango aceptable, el servidor enviará una respuesta que incluye un valor de Estado de 14 y un elemento Límite. Se puede incluir HeartbeatInterval o Wait en una solicitud de comando Sync, pero no ambas cosas. Si se incluyen ambos elementos, la respuesta del servidor contendrá un valor de Estado de 4.
