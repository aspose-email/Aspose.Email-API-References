## BodyPreference.AllOrNone property

Un cliente puede incluir varios elementos BodyPreference en una solicitud de comando con diferentes valores para el elemento Tipo (sección 2.2.2.22.4). De forma predeterminada, el servidor devuelve los datos truncados al tamaño solicitado por TruncationSize para el elemento Tipo que coincide con el formato de almacenamiento nativo del elemento Cuerpo del elemento. Pero, si el cliente también incluye el elemento AllOrNone con un valor de TRUE junto con el elemento TruncationSize, le está indicando al servidor que no devuelva una respuesta truncada para ese tipo cuando el tamaño (en bytes) de los datos disponibles excede el valor del elemento TruncationSize. Por ejemplo, un cliente puede usar estos dos elementos para indicar que no puede procesar datos parciales en formato de texto enriquecido (RTF) (un valor de elemento Tipo de 3). En este caso, si el cliente ha especificado varios elementos BodyPreference, el servidor selecciona el siguiente elemento BodyPreference que devolverá la cantidad máxima de cuerpo de texto al cliente. Suponga que el cliente especifica dos elementos BodyPreference. Si el elemento AllOrNone no está definido, el contenido truncado o no truncado se busca, sincroniza o recupera como si el valor se hubiera establecido en FALSE.
