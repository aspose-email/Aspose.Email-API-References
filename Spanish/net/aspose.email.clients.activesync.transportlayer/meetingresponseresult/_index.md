| [CalendarId](../../aspose.email.clients.activesync.transportlayer/meetingresponseresult/calendarid) { get; set; } | Especifica el ID del servidor del elemento del calendario. El valor del elemento CalendarId puede tener hasta 64 caracteres de longitud. El elemento CalendarId se incluye en la respuesta del comando MeetingResponse que se envía al cliente si la solicitud de reunión no se rechazó. Si la reunión se acepta o se acepta tentativamente, el servidor agrega un nuevo elemento al calendario y devuelve su ID de servidor en el elemento CalendarId en la respuesta. Si el cliente creó un elemento de calendario de reunión tentativo, el cliente actualiza ese elemento con la nueva ID del servidor. El cliente también cambia el estado ocupado de tentativo a ocupado. Cuando se acepta una reunión, el servidor también crea un nuevo elemento de calendario con el mismo ID de servidor. Esto significa que hay un conflicto que se resolverá la próxima vez que se sincronice el calendario. Si se rechaza la reunión, la respuesta no contiene un elemento CalendarId. |
