## SyncOptions.Conflict property

Especifica cómo resolver el conflicto que ocurre cuando se cambia un objeto tanto en el cliente como en el servidor. El valor especifica qué objeto (el objeto del cliente o el objeto del servidor) conservar si hay un conflicto. Si no se define el valor, el objeto del servidor reemplazará al objeto del cliente cuando ocurra un conflicto.

public BehaviorReplacement? Conflict { get; set; }
