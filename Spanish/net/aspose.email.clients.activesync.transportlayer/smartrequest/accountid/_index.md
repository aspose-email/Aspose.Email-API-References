## SmartRequest.AccountId property

Identifica la cuenta desde la que se envía un correo electrónico. Si AccountId es nulo, el servidor envía el correo electrónico utilizando la cuenta identificada por la configuración:PrimarySmtpAddress devuelta en la respuesta del comando Configuración. Si AccountId se incluye en la solicitud, el valor DEBE ser igual a uno de los valores del elemento settings:AccountId incluidos en la respuesta del comando Settings. El servidor DEBE validar que el valor del elemento AccountId proporcionado es válido para enviar correo electrónico. Se devuelve un valor de Estado de 166 si el valor del elemento AccountId no es válido. Se devuelve un valor de elemento de estado de 167 si la cuenta no admite el envío de correo electrónico. Nota El servidor envía el correo electrónico utilizando la cuenta especificada por AccountId, y no la cuenta especificada por From. El elemento AccountId no es compatible cuando la versión del protocolo es 12.1 o 14.0. Exchange 2007 devuelve el valor de estado 103 si el elemento AccountId se incluye en una solicitud de comando SendMail, una solicitud de comando SmartForward o una solicitud de comando SmartReply.
