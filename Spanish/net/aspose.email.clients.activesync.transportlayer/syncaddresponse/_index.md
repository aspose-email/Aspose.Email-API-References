El cliente incluye el elemento ClientId en la solicitud Agregar elemento que envía al servidor. La respuesta del servidor contiene un elemento Agregar que contiene el ID de cliente original y un nuevo ID de servidor que se asignó para el objeto, que reemplaza el ID de cliente como identificador de objeto permanente. El elemento ClientId es un identificador único que consta de hasta 64 dígitos y letras. El cliente genera este ID. El valor solo tiene que ser único para el dispositivo durante la duración de la solicitud de sincronización que agrega el objeto al servidor. El cliente almacena los ID de cliente hasta que la sesión de sincronización se completa correctamente, para facilitar la recuperación si falla el proceso de sincronización. |
