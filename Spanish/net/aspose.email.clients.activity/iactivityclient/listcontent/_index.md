## ListContent(string) {#listcontent}

Esta operación enumera el contenido actualmente disponible para su recuperación para el tipo de contenido especificado. El contenido es una agregación de acciones y eventos recolectados de múltiples servidores en múltiples centros de datos. El contenido se enumerará en el orden en que las agregaciones estén disponibles, pero no se garantiza que los eventos y acciones dentro de las agregaciones sean secuenciales. De forma predeterminada, cuando se omiten startTime y endTime, se devuelve el contenido disponible en las últimas 24 horas.

```csharp
public ContentInfo[] ListContent(string contentType)
```

| contentType | String | Indica el tipo de contenido. |

## ListContent(string, DateTime?, DateTime?) {#listcontent_1}

Esta operación enumera el contenido actualmente disponible para su recuperación para el tipo de contenido especificado. El contenido es una agregación de acciones y eventos recolectados de múltiples servidores en múltiples centros de datos. El contenido se enumerará en el orden en que las agregaciones estén disponibles, pero no se garantiza que los eventos y acciones dentro de las agregaciones sean secuenciales.

```csharp
public ContentInfo[] ListContent(string contentType, DateTime? startTime, DateTime? endTime)
```

| Parámetro | Escribe | Descripción |
| --- | --- | --- |
| contentType | String | Indica el tipo de contenido. |
| startTime | Nullable`1 | Horas de fecha y hora (UTC) opcionales que indican el intervalo de tiempo del contenido que se devolverá, en función de cuándo estuvo disponible el contenido. El rango de tiempo es inclusivo con respecto a startTime (startTime menos o igual contenido creado), de modo que los intervalos de tiempo incrementales que no se superponen se pueden usar para navegar a través del contenido disponible. Ambos (startTime y endTime) deben especificarse (o omitirse ambos) ). El intervalo de tiempo no debe tener más de 24 horas de diferencia, con la hora de inicio no más de 7 días antes. |
| endTime | Nullable`1 | Horas de fecha y hora (UTC) opcionales que indican el intervalo de tiempo del contenido que se devolverá, en función de cuándo estuvo disponible el contenido. El rango de tiempo es exclusivo con respecto a endTime (contenido creado menos endTime), de modo que los intervalos de tiempo incrementales que no se superponen se pueden usar para navegar a través del contenido disponible. Ambos (startTime y endTime) deben especificarse (o ambos omitirse) El rango de tiempo no debe tener más de 24 horas de diferencia, con la hora de inicio no más de 7 días antes. |
