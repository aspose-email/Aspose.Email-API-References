| [Range](../../aspose.email.clients.activesync.transportlayer/searchresponsestore/range) { get; set; } | Spécifie le nombre d'entrées correspondantes renvoyées. Le format de la valeur de l'élément Range se présente sous la forme d'un spécificateur d'index de base zéro, formé d'un zéro, d'un trait d'union et d'une autre valeur numérique : "mn". Le m indique l'indice le plus bas d'un tableau de base zéro qui contiendrait les éléments. Le n indique l'indice le plus élevé d'un tableau de base zéro qui contiendrait les éléments. Par exemple, une valeur d'élément Range de 0 à 9 indique 10 éléments et 0 à 10 indique 11 éléments. Une valeur d'élément Range de 0 à 0 indique 1 élément. Si Range est nul, la valeur Range par défaut pour chaque type de magasin est utilisée. Le tableau suivant identifie les valeurs de plage par défaut et les résultats maximum renvoyés pour chaque type de magasin : Mailbox - Valeur de plage par défaut : 0-99 - Résultats maximum renvoyés : 100 DocumentLibrary - Valeur de plage par défaut : 0-999 - Résultats maximum renvoyés : 1000 GAL - Valeur de plage par défaut : 0-99 - Résultats maximum renvoyés : 100 Si la valeur de plage spécifiée dans la demande dépasse la valeur de plage par défaut, une valeur d'état de 12 est renvoyée pour indiquer que la plage maximale a été dépassée. Dans la réponse de la commande Search, la propriété Total indique une estimation du nombre total d'entrées correspondant à la valeur Query. Les résultats de la recherche sont stockés dans un dossier de recherche sur le serveur. De cette façon, lorsqu'un client revient avec la même requête mais une nouvelle plage de lignes, les lignes sont extraites du jeu de résultats actuellement stocké dans le dossier de recherche. L'ensemble des résultats n'a pas besoin d'être reconstruit. |

| [Total](../../aspose.email.clients.activesync.transportlayer/searchresponsestore/total) { get; set; } | Fournit une estimation du nombre total d'entrées de boîte aux lettres correspondant à la valeur de la requête de recherche. La valeur du total n'est pas toujours égale au nombre d'entrées renvoyées. Pour déterminer le nombre d'entrées renvoyées par la commande Rechercher, utilisez la valeur Plage. Le total indique le nombre d'entrées disponibles. Dans les cas où tous les résultats sont renvoyés dans la réponse XML, la valeur de l'élément Total est supérieure d'une unité à la valeur d'index de fin fournie dans l'élément Range. Par exemple, si la commande Rechercher renvoie 15 entrées, la valeur de l'élément Range est comprise entre 0 et 14, tandis que la valeur du Total est 15. Le Total est utilisé par les clients pour déterminer si d'autres entrées correspondantes ont été trouvées dans la boîte aux lettres que celles renvoyées par la commande Rechercher. Par exemple, un client peut effectuer une recherche initiale et spécifier une plage demandée de 0 à 4 (renvoyer 5 entrées maximum). Si l'élément Total indique qu'il existe en fait 25 éléments correspondants, le client peut alors permettre à l'utilisateur de récupérer les résultats complets. |
