## SearchResponseStore.Range property

Spécifie le nombre d'entrées correspondantes renvoyées. Le format de la valeur de l'élément Range se présente sous la forme d'un spécificateur d'index de base zéro, formé d'un zéro, d'un trait d'union et d'une autre valeur numérique : "mn". Le m indique l'indice le plus bas d'un tableau de base zéro qui contiendrait les éléments. Le n indique l'indice le plus élevé d'un tableau de base zéro qui contiendrait les éléments. Par exemple, une valeur d'élément Range de 0 à 9 indique 10 éléments et 0 à 10 indique 11 éléments. Une valeur d'élément Range de 0 à 0 indique 1 élément. Si Range est nul, la valeur Range par défaut pour chaque type de magasin est utilisée. Le tableau suivant identifie les valeurs de plage par défaut et les résultats maximum renvoyés pour chaque type de magasin : Mailbox - Valeur de plage par défaut : 0-99 - Résultats maximum renvoyés : 100 DocumentLibrary - Valeur de plage par défaut : 0-999 - Résultats maximum renvoyés : 1000 GAL - Valeur de plage par défaut : 0-99 - Résultats maximum renvoyés : 100 Si la valeur de plage spécifiée dans la demande dépasse la valeur de plage par défaut, une valeur d'état de 12 est renvoyée pour indiquer que la plage maximale a été dépassée. Dans la réponse de la commande Search, la propriété Total indique une estimation du nombre total d'entrées correspondant à la valeur Query. Les résultats de la recherche sont stockés dans un dossier de recherche sur le serveur. De cette façon, lorsqu'un client revient avec la même requête mais une nouvelle plage de lignes, les lignes sont extraites du jeu de résultats actuellement stocké dans le dossier de recherche. L'ensemble des résultats n'a pas besoin d'être reconstruit.
