| [WindowSize](../../aspose.email.clients.activesync.transportlayer/syncrequest/windowsize) { get; set; } | Spécifie un nombre maximal d'éléments modifiés dans une collection ou une demande qui DEVRAIT être inclus dans la réponse de synchronisation. La valeur maximale de WindowSize est 512. Cependant, si WindowSize est défini sur 512, le serveur peut envoyer des messages de réponse Sync contenant moins de 512 mises à jour. Si le serveur n'envoie pas toutes les mises à jour dans un seul message, le message de réponse Sync contient le MoreAvailable, qui indique qu'il existe des mises à jour supplémentaires sur le serveur à télécharger sur le client. Le WindowSize n'apparaît que dans les demandes envoyées au serveur depuis le client. Si WindowSize est omis, le serveur se comporte comme si un élément WindowSize avec une valeur de 100 était soumis. Le serveur interprète la valeur 0 (zéro) et les valeurs supérieures à 512 comme 512. |
