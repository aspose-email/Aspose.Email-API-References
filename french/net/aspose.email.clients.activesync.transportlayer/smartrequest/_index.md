| [AccountId](../../aspose.email.clients.activesync.transportlayer/smartrequest/accountid) { get; set; } | Identifie le compte à partir duquel un e-mail est envoyé. Si AccountId est nul, le serveur envoie l'e-mail à l'aide du compte identifié par les paramètres :PrimarySmtpAddress renvoyé dans la réponse de la commande Settings. Si AccountId est inclus dans la requête, la valeur DOIT être égal à l'une des valeurs de l'élément settings:AccountId incluses dans la réponse de la commande Settings. Le serveur DOIT valider que la valeur de l'élément AccountId fournie est valide pour l'envoi d'e-mails. Une valeur de statut de 166 est renvoyée si la valeur de l'élément AccountId n'est pas valide. Une valeur d'élément Status de 167 est renvoyée si le compte ne prend pas en charge l'envoi d'e-mails. Remarque Le serveur envoie l'e-mail en utilisant le compte spécifié par AccountId, et non le compte spécifié par From. L'élément AccountId n'est pas pris en charge lorsque la version du protocole est 12.1 ou 14.0. Exchange 2007 renvoie la valeur Status 103 si l'élément AccountId est inclus dans une demande de commande SendMail, une demande de commande SmartForward ou une demande de commande SmartReply. |
| [ClientId](../../aspose.email.clients.activesync.transportlayer/smartrequest/clientid) { get; set; } | Spécifie l'ID de message unique (MID) du client. La valeur ClientId peut être un simple compteur incrémenté à chaque nouveau message. |
| [Mime](../../aspose.email.clients.activesync.transportlayer/smartrequest/mime) { get; set; } | Contient le message encodé MIME. Si le message contient une demande de réunion, l'élément Mime contient les détails de la réunion dans Format iCalendar [MS-OXCICAL] ou format TNEF (Transport Neutral Encapsulation Format) [MS-OXTNEF]. Comme spécifié dans la section 3.4 de la [RFC2447], les demandes de réunion iCalendar ont un type de contenu de "texte/calendrier" avec le paramètre de méthode défini sur "REQUEST". |
| [ReplaceMime](../../aspose.email.clients.activesync.transportlayer/smartrequest/replacemime) { get; set; } | Spécifie si le client envoie le message entier. Lorsque ReplaceMime est FALSE, le client DOIT ajouter le corps du message d'origine en pièces jointes au message sortant. Le client peut utiliser cette propriété pour indiquer si le message a été modifié en ligne ou si le message avait un texte de réponse/transfert ajouté au message source. Si le ReplaceMime est VRAI, le message a été modifié. |
