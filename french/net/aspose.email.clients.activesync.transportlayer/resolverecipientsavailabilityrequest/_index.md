## ResolveRecipientsAvailabilityRequest class

Indique au serveur que des données de disponibilité sont demandées par le client et identifie l'heure de début et l'heure de fin des données de disponibilité à récupérer. La disponibilité n'est pas prise en charge lorsque la version du protocole est 12.1.

## Propriétés

| Nom | La description |
| --- | --- |
| [EndTime](../../aspose.email.clients.activesync.transportlayer/resolverecipientsavailabilityrequest/endtime) { get; set; } | Identifie la fin de la période de disponibilité demandée par le client. EndTime n'est pas pris en charge lorsque la version du protocole est 12.1. Si le client envoie une valeur EndTime non valide, le serveur renvoie une valeur Status de 5 dans la réponse de la commande ResolveRecipients. Si aucun EndTime n'est inclus dans une demande de disponibilité, le serveur utilise une valeur d'heure de fin par défaut de sept jours après la valeur StartTime. Si la valeur EndTime spécifiée dans la demande est inférieure à la valeur StartTime plus 30 minutes, ou si la durée comprise entre la valeur StartTime et la valeur EndTime est supérieure à un nombre spécifique de jours, le serveur renvoie une valeur Status de 5 dans la réponse de la commande ResolveRecipients. Exchange 2010 et Exchange 2013 utilisent 42 jours ; Exchange 2007 SP1 utilise 62 jours. |
| [StartTime](../../aspose.email.clients.activesync.transportlayer/resolverecipientsavailabilityrequest/starttime) { get; set; } | Identifie le début de la période de temps libre/occupé demandée par le client. Le StartTime n'est pas pris en charge lorsque la version du protocole est 12.1. Si la disponibilité est incluse dans la demande, la demande DOIT également inclure le StartTime et EndTime. Si le client envoie une valeur StartTime non valide, le serveur renvoie une valeur Status de 5 dans la réponse de la commande ResolveRecipients. |
