## BodyPreference.AllOrNone property

Un client peut inclure plusieurs éléments BodyPreference dans une demande de commande avec différentes valeurs pour l'élément Type (section 2.2.2.22.4). Par défaut, le serveur renvoie les données tronquées à la taille demandée par TruncationSize pour l'élément Type qui correspond au format de stockage natif de l'élément Body de l'élément. Mais, si le client inclut également l'élément AllOrNone avec une valeur TRUE avec l'élément TruncationSize, il demande au serveur de ne pas renvoyer de réponse tronquée pour ce type lorsque la taille (en octets) des données disponibles dépasse la valeur de l'élément TruncationSize. Par exemple, un client peut utiliser ces deux éléments pour signifier qu'il ne peut pas traiter des données RTF (Rich Text Format) partielles (une valeur d'élément Type de 3). Dans ce cas, si le client a spécifié plusieurs éléments BodyPreference, le serveur sélectionne l'élément BodyPreference suivant qui renverra la quantité maximale de corps de texte au client. Supposons que le client spécifie deux éléments BodyPreference. Si l'élément AllOrNone n'est pas défini, le contenu tronqué ou non tronqué est recherché, synchronisé ou récupéré comme si la valeur était définie sur FALSE.
