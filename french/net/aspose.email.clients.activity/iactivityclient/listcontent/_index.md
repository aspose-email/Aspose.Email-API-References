## ListContent(string) {#listcontent}

Cette opération répertorie le contenu actuellement disponible pour récupération pour le type de contenu spécifié. Le contenu est une agrégation d'actions et d'événements récoltés à partir de plusieurs serveurs dans plusieurs centres de données. Le contenu sera répertorié dans l'ordre dans lequel les agrégations deviennent disponibles, mais les événements et les actions au sein des agrégations ne sont pas garantis comme étant séquentiels. Par défaut, lorsque startTime et endTime sont omis, le contenu disponible au cours des dernières 24 heures est renvoyé.

```csharp
public ContentInfo[] ListContent(string contentType)
```

| contentType | String | Indique le type de contenu. |

## ListContent(string, DateTime?, DateTime?) {#listcontent_1}

Cette opération répertorie le contenu actuellement disponible pour récupération pour le type de contenu spécifié. Le contenu est une agrégation d'actions et d'événements récoltés à partir de plusieurs serveurs dans plusieurs centres de données. Le contenu sera répertorié dans l'ordre dans lequel les agrégations deviennent disponibles, mais les événements et les actions au sein des agrégations ne sont pas garantis comme étant séquentiels.

```csharp
public ContentInfo[] ListContent(string contentType, DateTime? startTime, DateTime? endTime)
```

| Paramètre | Taper | La description |
| --- | --- | --- |
| contentType | String | Indique le type de contenu. |
| startTime | Nullable`1 | Datetimes (UTC) facultatif indiquant la plage horaire du contenu à renvoyer, en fonction du moment où le contenu est devenu disponible. La plage de temps est inclusive par rapport à startTime (startTime inférieur ou égal à contentCreated), afin que des intervalles de temps incrémentiels sans chevauchement puissent être utilisés pour parcourir le contenu disponible. Les deux (startTime et endTime) doivent être spécifiés (ou les deux omis ). La plage horaire ne doit pas être espacée de plus de 24 heures, l'heure de début ne dépassant pas 7 jours dans le passé. |
| endTime | Nullable`1 | Datetimes (UTC) facultatif indiquant la plage horaire du contenu à renvoyer, en fonction du moment où le contenu est devenu disponible. La plage de temps est exclusive par rapport à endTime (contentCreated moins endTime), afin que des intervalles de temps incrémentiels sans chevauchement puissent être utilisés pour parcourir le contenu disponible. Les deux (startTime et endTime) doivent être spécifiés (ou les deux omis) La plage horaire ne doit pas être espacée de plus de 24 heures, l'heure de début ne dépassant pas 7 jours dans le passé. |
