## IAsyncImapClient.ResumeMonitoringAsync method

Reprend la surveillance des modifications de message pour le dossier spécifié. Contrairement à la méthode StartMonitoring, elle trouvera toutes les modifications de boîte aux lettres manquantes et appellera le rappel correspondant.

```csharp
public Task ResumeMonitoringAsync(ImapMonitoringEventHandler callback, 
    ImapMonitoringErrorEventHandler errorCallback, IImapMonitoringState monitoringState, 
    CancellationToken token = default)
```

| Paramètre | Taper | La description |
| --- | --- | --- |
| callback | ImapMonitoringEventHandler | La fonction de rappel pour surveiller le fonctionnement. |
| errorCallback | ImapMonitoringErrorEventHandler | La fonction de rappel pour surveiller la gestion des erreurs. La surveillance du dossier spécifié est arrêtée lorsque ce rappel est appelé. Le rappel fournit également un détenteur d'état afin que la surveillance du dossier puisse être reprise en utilisant[`ResumeMonitoringAsync`](../../imapclient/resumemonitoringasync) méthode. |
| monitoringState | IImapMonitoringState | L'état de surveillance à partir duquel reprendre la surveillance du dossier. Peut être récupéré à partir de[`MonitoringState`](../../imapmonitoringerroreventargs/monitoringstate) . |
| token | CancellationToken | Le jeton d'annulation. |
