## DSNAction enumeration

Indique l'action effectuée par le Reporting-MTA à la suite de sa tentative de livraison du message.

```csharp
public enum DSNAction
```

### Valeurs

| Nom | Évaluer | La description |
| --- | --- | --- |
| None | `0` | Aucune des actions. |
| Delayed | `1` | Indique que le MTA de rapport n'a jusqu'à présent pas été en mesure de remettre ou de relayer le message, mais qu'il continuera à tenter de le faire. Des messages de notification supplémentaires peuvent être émis lorsque le message est encore retardé ou livré avec succès, ou si les tentatives de livraison sont ultérieurement abandonnées. |
| Delivered | `2` | Indique que le message a été livré avec succès à l'adresse du destinataire spécifiée par l'expéditeur, qui inclut la "livraison" à un exploseur de liste de diffusion. Cela n'indique pas que le message a été lu. Il s'agit d'un état terminal et aucun autre DSN ne doit être attendu pour ce destinataire. |
| Expanded | `3` | Indique que le message a été livré avec succès à l'adresse du destinataire spécifiée par l'expéditeur, et transmis par le Reporting-MTA au-delà de cette destination vers plusieurs adresses de destinataires supplémentaires. Une valeur d'action "étendu" diffère de Livré en ce que "étendu" n'est pas un état terminal. D'autres notifications "échec" et/ou "retard" peuvent être fournies. |
| Failed | `4` | Indique que le message n'a pas pu être remis au destinataire. Le MTA de rapport a abandonné toute tentative de remise du message à ce destinataire. Aucune autre notification ne devrait être attendue. |
| Relayed | `5` | Indique que le message a été relayé ou transféré dans un environnement qui n'accepte pas la responsabilité de générer des DSN lors de la livraison réussie. Cette valeur d'action NE DEVRAIT PAS être utilisée sauf si l'expéditeur a demandé une notification de livraison réussie pour ce destinataire. |
