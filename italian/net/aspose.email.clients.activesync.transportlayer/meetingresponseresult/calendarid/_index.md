## MeetingResponseResult.CalendarId property

Specifica l'ID server dell'elemento del calendario. Il valore dell'elemento CalendarId può avere una lunghezza massima di 64 caratteri. L'elemento CalendarId è incluso nella risposta al comando MeetingResponse inviata al client se la convocazione di riunione non è stata rifiutata. Se la riunione viene accettata o accettata provvisoriamente, il server aggiunge un nuovo elemento al calendario e restituisce il suo ID server nell'elemento CalendarId nella risposta. Se il client ha creato un elemento del calendario della riunione provvisorio, il client aggiorna tale elemento con il nuovo ID server. Il client cambia anche lo stato di occupato da provvisorio a occupato. Quando una riunione viene accettata, il server crea anche un nuovo elemento del calendario con lo stesso ID server. Ciò significa che c'è un conflitto che verrà risolto alla successiva sincronizzazione del calendario. Se la riunione viene rifiutata, la risposta non contiene un elemento CalendarId.
