## ResolveRecipientsAvailabilityRequest.EndTime property

Identifica la fine dell'intervallo di tempo libero/occupato richiesto dal client. EndTime non è supportato quando la versione del protocollo è 12.1. Se il client invia un valore EndTime non valido, il server restituisce un valore di stato di 5 in il comando ResolveRecipients response. Se in una richiesta di disponibilità non è incluso EndTime, il server utilizza un valore di ora di fine predefinito di sette giorni dopo il valore di StartTime. Se il valore EndTime specificato nella richiesta è inferiore al valore StartTime più 30 minuti o la durata compresa tra il valore StartTime e il valore EndTime è maggiore di un numero specifico di giorni, il server restituisce un valore Status di 5 in il comando ResolveRecipients response. Exchange 2010 ed Exchange 2013 utilizzano 42 giorni; Exchange 2007 SP1 utilizza 62 giorni.
