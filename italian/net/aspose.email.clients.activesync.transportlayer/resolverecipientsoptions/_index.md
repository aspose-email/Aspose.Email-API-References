| [Availability](../../aspose.email.clients.activesync.transportlayer/resolverecipientsoptions/availability) { get; set; } | Indica al server che i dati sulla disponibilità sono richiesti dal client e identifica l'ora di inizio e l'ora di fine dei dati sulla disponibilità da recuperare. Quando la disponibilità è inclusa in una richiesta ResolveRecipients, il server recupera la disponibilità informazioni per gli utenti identificati negli elementi To inclusi nella richiesta e restituisce le informazioni sulla disponibilità in MergedFreeBusy nella risposta. Se l'elemento Availability è incluso nella richiesta ResolveRecipients, la richiesta DEVE includere anche un valore StartTime e un valore EndTime validi. Quando il server analizza la richiesta, il server risolve prima i destinatari identificati dagli elementi A, quindi determina le informazioni sulla disponibilità degli utenti per l'intervallo di tempo specificato, prima di restituire i dati sulla disponibilità in MergedFreeBusy. |
| [CertificateRetrieval](../../aspose.email.clients.activesync.transportlayer/resolverecipientsoptions/certificateretrieval) { get; set; } | Specifica se i certificati S/MIME DEVONO essere restituiti dal server per ogni destinatario risolto. |
| [MaxAmbiguousRecipients](../../aspose.email.clients.activesync.transportlayer/resolverecipientsoptions/maxambiguousrecipients) { get; set; } | Limita il numero di suggerimenti restituiti per ciascun nodo destinatario ambiguo nella risposta. Il valore di MaxAmbiguousRecipients è limitato a un intervallo compreso tra 0 e 9999. Ogni nodo destinatario ambiguo riceve solo questo numero di suggerimenti e non di più. Il conteggio dei destinatari, restituito in RecipientCount, può essere utilizzato dal client per determinare il numero totale di suggerimenti disponibili per quel destinatario. |
| [MaxCertificates](../../aspose.email.clients.activesync.transportlayer/resolverecipientsoptions/maxcertificates) { get; set; } | Limita il numero totale di certificati restituiti dal server. Il valore di MaxCertificates è limitato a un intervallo compreso tra 0 e 9999. Questo limite garantisce che nessun singolo destinatario riceva una serie incompleta di certificati. Se viene raggiunto il limite MaxCertificates durante l'enumerazione dei certificati per un elenco di indirizzi, tale elenco di indirizzi non restituirà alcun certificato e viene restituito un valore di stato pari a 8. |
| [Picture](../../aspose.email.clients.activesync.transportlayer/resolverecipientsoptions/picture) { get; set; } | Indica che il client sta richiedendo che le foto dei contatti vengano restituite nella risposta del server. L'immagine non è supportata quando la versione del protocollo è 12.1 o 14.0. Contiene i dati relativi alle foto dei contatti. |
