| [AccountId](../../aspose.email.clients.activesync.transportlayer/smartrequest/accountid) { get; set; } | Identifica l'account da cui viene inviata un'e-mail. Se AccountId è nullo, il server invia l'e-mail utilizzando l'account identificato dalle impostazioni:PrimarySmtpAddress restituito nella risposta del comando Impostazioni. Se AccountId è incluso nella richiesta, il valore DEVE essere uguale a una delle impostazioni: valori dell'elemento AccountId inclusi nella risposta del comando Impostazioni. Il server DEVE convalidare che il valore dell'elemento AccountId fornito è valido per l'invio di e-mail. Viene restituito un valore di stato pari a 166 se il valore dell'elemento AccountId non è valido. Viene restituito un valore dell'elemento Status di 167 se l'account non supporta l'invio di e-mail. Nota Il server invia l'e-mail utilizzando l'account specificato da AccountId e non l'account specificato da From. L'elemento AccountId non è supportato quando la versione del protocollo è 12.1 o 14.0. Exchange 2007 restituisce il valore di stato 103 se l'elemento AccountId è incluso in una richiesta di comando SendMail, una richiesta di comando SmartForward o una richiesta di comando SmartReply. |
| [ClientId](../../aspose.email.clients.activesync.transportlayer/smartrequest/clientid) { get; set; } | Specifica l'ID messaggio univoco (MID) del client. Il valore ClientId può avere una lunghezza massima di 40 caratteri e DEVE essere univoco per ogni messaggio, poiché il server utilizzerà il valore ClientId per identificare i messaggi duplicati. Il valore ClientId può essere un semplice contatore incrementato per ogni nuovo messaggio. |
| [Mime](../../aspose.email.clients.activesync.transportlayer/smartrequest/mime) { get; set; } | Contiene il messaggio con codifica MIME. Il contenuto Mime viene trasferito come BLOB opaco all'interno dei tag WBXML, come specificato in [WBXML1.2]. Se il messaggio contiene una convocazione di riunione, l'elemento Mime contiene i dettagli della riunione in Formato iCalendar [MS-OXCICAL] o formato TNEF (Transport Neutral Encapsulation Format) [MS-OXTNEF]. Come specificato in [RFC2447] sezione 3.4, le convocazioni di riunione di iCalendar hanno un tipo di contenuto di "testo/calendario" con il parametro del metodo impostato su "RICHIESTA". |
| [ReplaceMime](../../aspose.email.clients.activesync.transportlayer/smartrequest/replacemime) { get; set; } | Specifica se il client sta inviando l'intero messaggio. Quando il ReplaceMime è FALSE, il client DEVE aggiungere il corpo del messaggio originale come allegati al messaggio in uscita. Il client può utilizzare questa proprietà per indicare se il messaggio è stato modificato in linea o se il messaggio ha un testo di risposta/inoltro anteposto al messaggio di origine. Se ReplaceMime è TRUE, il messaggio è stato modificato. |
