## BodyPreference.AllOrNone property

Un client può includere più elementi BodyPreference in una richiesta di comando con valori diversi per l'elemento Type (sezione 2.2.2.22.4). Per impostazione predefinita, il server restituisce i dati troncati alla dimensione richiesta da TruncationSize per l'elemento Type che corrisponde al formato di archiviazione nativo dell'elemento Body dell'elemento. Ma, se il client include anche l'elemento AllOrNone con un valore di TRUE insieme all'elemento TruncationSize, sta indicando al server di non restituire una risposta troncata per quel tipo quando la dimensione (in byte) dei dati disponibili supera il valore dell'elemento TruncationSize. Ad esempio, un client può utilizzare questi due elementi per indicare che non può elaborare dati RTF (Rich Text Format) parziali (un valore di elemento Type pari a 3). In questo caso, se il client ha specificato più elementi BodyPreference, il server seleziona l'elemento BodyPreference successivo che restituirà la quantità massima di corpo del testo al client. Si supponga che il client specifichi due elementi BodyPreference. Se l'elemento AllOrNone non è definito, il contenuto troncato o non troncato viene cercato, sincronizzato o recuperato come se il valore fosse impostato su FALSE.
