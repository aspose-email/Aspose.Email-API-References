## BackupSettings.NumberOfAttemptsToRrepeat property

Ottiene o imposta il valore che definisce il numero di tentativi per ripetere l'operazione non riuscita Nel caso in cui alcuni comandi IMAP all'interno dell'operazione di backup restituiscano risultati non riusciti, il client IMAP tenta di ripetere nuovamente questa operazione in base al numero di volte definito. Ad esempio se l'operazione FETCH restituisce error AE_1_1_0243 FETCH 219 (BODY) AE_1_1_0243 NO[NON DISPONIBILE] Il servizio FETCH non è temporaneamente disponibile Il client tenta di eseguirlo di nuovo.

public int NumberOfAttemptsToRrepeat { get; set; }
