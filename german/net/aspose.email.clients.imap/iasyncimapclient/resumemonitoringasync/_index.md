## IAsyncImapClient.ResumeMonitoringAsync method

Nimmt die Überwachung von Nachrichtenänderungen für den angegebenen Ordner wieder auf. Im Gegensatz zur StartMonitoring-Methode findet sie alle fehlenden Postfachänderungen und ruft den Rückruf für sie auf.

```csharp
public Task ResumeMonitoringAsync(ImapMonitoringEventHandler callback, 
    ImapMonitoringErrorEventHandler errorCallback, IImapMonitoringState monitoringState, 
    CancellationToken token = default)
```

| Parameter | Typ | Beschreibung |
| --- | --- | --- |
| callback | ImapMonitoringEventHandler | Die Callback-Funktion zur Überwachung des Betriebs. |
| errorCallback | ImapMonitoringErrorEventHandler | Die Callback-Funktion zur Überwachung der Fehlerbehandlung. Die Überwachung des angegebenen Ordners wird gestoppt, wenn dieser Callback aufgerufen wird. Der Callback stellt auch einen Zustandshalter bereit, damit die Ordnerüberwachung fortgesetzt werden kann mit[`ResumeMonitoringAsync`](../../imapclient/resumemonitoringasync) Methode. |
| monitoringState | IImapMonitoringState | Der Überwachungsstatus, ab dem die Ordnerüberwachung fortgesetzt werden soll. Kann abgerufen werden[`MonitoringState`](../../imapmonitoringerroreventargs/monitoringstate) . |
| token | CancellationToken | Das Abbruchtoken. |
