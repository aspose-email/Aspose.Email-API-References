## EASProvisionDoc.DevicePasswordHistory property

Gibt die Mindestanzahl von zuvor verwendeten Passwörtern an, die gespeichert werden, um eine Wiederverwendung durch den Client zu verhindern. Gültige Werte sind unten aufgeführt: = 0 – Eine Speicherung von zuvor verwendeten Passwörtern ist nicht erforderlich. &gt; 0 – Die Mindestanzahl von zuvor verwendeten Passwörtern muss sein gespeichert. Wenn DevicePasswordHistory null ist, dann SOLLTE ein Client diesen Wert als 0 behandeln. Wenn der Wert von DevicePasswordEnabled auf TRUE gesetzt ist, verbietet der Client dem Benutzer, ein gespeichertes vorheriges Passwort zu verwenden, nachdem ein Passwort abgelaufen ist. Wenn der Wert von DevicePasswordEnabled auf FALSE gesetzt ist, SOLLTE der Client diese Eigenschaft ignorieren.
