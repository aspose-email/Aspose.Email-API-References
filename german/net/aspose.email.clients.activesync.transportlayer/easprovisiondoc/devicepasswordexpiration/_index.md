## EASProvisionDoc.DevicePasswordExpiration property

Gibt die maximale Anzahl von Tagen an, bis ein Kennwort abläuft. DevicePasswordExpiration kann null sein, was darauf hinweist, dass keine Kennwortablaufrichtlinie festgelegt ist. Gültige Werte sind unten aufgeführt: = 0 – Kennwörter laufen nicht ab. &gt; 0 – Kennwörter laufen ab in der angegebenen maximalen Anzahl von Tagen. Wenn DevicePasswordExpiration null ist, SOLLTE ein Client diesen Wert als 0 behandeln. Wenn der Wert von DevicePasswordEnabled auf FALSE gesetzt ist, SOLLTE der Client diese Eigenschaft ignorieren.
