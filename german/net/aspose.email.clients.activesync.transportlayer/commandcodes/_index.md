## CommandCodes enumeration

Die folgende Tabelle enthält die numerischen Codes, die den ActiveSync-Befehlen entsprechen. Der numerische Code wird im Feld Befehlscode des base64-codierten URI verwendet, um den Befehl anzugeben.

```csharp
public enum CommandCodes
```

### Werte

| Name | Wert | Beschreibung |
| --- | --- | --- |
| Undefined | `-3` | Befehlscode ist nicht definiert. |
| Options | `-2` | Der Befehl OPTIONS wird verwendet, um festzustellen, welche Protokollversionen unterstützt werden und welche Protokollbefehle auf dem Server unterstützt werden. |
| Autodiscover | `-1` | Der Autodiscover-Befehl erleichtert die Erkennung von Core-Kontokonfigurationsinformationen , indem die SMTP-Adresse (Simple Mail Transfer Protocol) des Benutzers als primäre Eingabe verwendet wird. |
| Sync | `0` | Synchronisiert Änderungen in einem Ordner zwischen dem Client und dem Server. |
| SendMail | `1` | Sendet Mail an den Server. Dieser Befehl wird im URI des HTTP-POST-Befehls ausgegeben und enthält keinen XML-Hauptteil. Der Textkörper enthält stattdessen die MIME-Nachricht. |
| SmartForward | `2` | Leitet ein Message-Objekt weiter, ohne das vollständige Message-Objekt vom Server abzurufen. |
| SmartReply | `3` | antwortet auf ein Message-Objekt, ohne das vollständige Message-Objekt vom Server abzurufen. |
| GetAttachment | `4` | Ruft einen E-Mail-Anhang vom Server ab. |
| FolderSync | `9` | Synchronisiert die Ordnerhierarchie, aber nicht die Elemente in den Ordnern. |
| FolderCreate | `10` | Erstellt einen E-Mail-, Kalender- oder Kontaktordner auf dem Server. |
| FolderDelete | `11` | Löscht einen Ordner vom Server. |
| FolderUpdate | `12` | Verschiebt einen Ordner von einem Speicherort an einen anderen auf dem Server und wird zum Umbenennen von Ordnern verwendet. |
| MoveItems | `13` | Verschiebt Elemente von einem Ordner in einen anderen. |
| GetItemEstimate | `14` | Ruft eine Schätzung der Anzahl der Elemente in einem synchronisierten Ordner ab. |
| MeetingResponse | `15` | Wird verwendet, um eine Besprechungsanfrage im Posteingangsordner des Benutzers anzunehmen, vorläufig anzunehmen oder abzulehnen. |
| Search | `16` | Sucht und ruft Informationen zu Kontakten und Empfängern in der globalen Adressliste ab. |
| Settings | `17` | Unterstützt das Abrufen und Festlegen globaler Eigenschaften wie Abwesenheit (OOF) und Geräteinformationen. |
| Ping | `18` | Fordert an, dass der Server bestimmte Ordner auf Änderungen überwacht, die eine erneute Synchronisierung des Clients erfordern würden. |
| ItemOperations | `19` | Identifiziert den Hauptteil der Anfrage oder Antwort als einen Satz von Befehlen enthaltend, die auf Items wirken. |
| Provision | `20` | Ruft die vom Serveradministrator festgelegten Sicherheitsrichtlinieneinstellungen ab, z. B. die Mindestkennwortlänge des Benutzers. |
| ResolveRecipients | `21` | Löst eine Liste der bereitgestellten Empfänger auf und ruft optional ihre S/MIME-Zertifikate ab, damit Clients verschlüsselte Nachrichten senden können. |
| ValidateCert | `22` | Validiert ein Zertifikat, das über eine S/MIME-Mail empfangen wurde. |
