## SearchOptions.Range property

Gibt die maximale Anzahl der zurückzugebenden übereinstimmenden Einträge an. Das Format des Bereichselementwerts hat die Form eines nullbasierten Indexbezeichners, der aus einer Null, einem Bindestrich und einem weiteren numerischen Wert besteht: „mn“. Das m gibt den niedrigsten Index eines nullbasierten Arrays an, das die Elemente enthalten würde. Das n gibt den höchsten Index eines nullbasierten Arrays an, das die Elemente enthalten würde. Ein Bereichselementwert von 0–9 gibt beispielsweise 10 Elemente an, und 0–10 gibt 11 Elemente an. Ein Range-Elementwert von 0–0 gibt 1 Element an. Wenn Range null ist, wird der Standard-Range-Wert für jeden Geschäftstyp verwendet. Die folgende Tabelle identifiziert die Standardbereichswerte und maximalen Ergebnisse, die für jeden Geschäftstyp zurückgegeben werden: Mailbox - Standardbereichswert: 0-99 - Maximale zurückgegebene Ergebnisse: 100 DocumentLibrary - Standardbereichswert: 0-999 - Maximale zurückgegebene Ergebnisse : 1000 GAL - Standardbereichswert: 0-99 - Maximal zurückgegebene Ergebnisse: 100 Wenn der in der Anfrage angegebene Bereichswert den Standardbereichswert überschreitet, wird ein Statuswert von 12 zurückgegeben, um anzuzeigen, dass der maximale Bereich überschritten wurde. In der Antwort auf den Suchbefehl gibt die Eigenschaft Gesamt eine Schätzung der Gesamtzahl der Einträge an, die mit dem Abfragewert übereinstimmten. Suchergebnisse werden in einem Suchordner auf dem Server gespeichert. Auf diese Weise werden, wenn ein Client mit derselben Abfrage, aber einem neuen Zeilenbereich zurückkommt, Zeilen aus der Ergebnismenge abgerufen, die derzeit im Suchordner gespeichert ist. Die gesamte Ergebnismenge muss nicht neu erstellt werden.
