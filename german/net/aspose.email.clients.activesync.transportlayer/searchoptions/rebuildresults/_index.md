## SearchOptions.RebuildResults property

Zwingt den Server, den Suchordner (2) neu zu erstellen, der einer bestimmten Abfrage entspricht. Die Suchergebnisse (d. h. die Ergebnismenge) werden in einem Suchordner auf dem Server gespeichert. Auf diese Weise werden, wenn ein Client mit derselben Abfrage, aber einem neuen Zeilenbereich zurückkommt, Zeilen aus der Ergebnismenge abgerufen, die derzeit im Suchordner gespeichert ist. Der gesamte Ergebnissatz muss nicht neu erstellt werden. Der Suchordner bleibt unverändert, bis der Client einen der folgenden Schritte ausführt, um den Ergebnissatz zu aktualisieren: - Sendet eine Suchanforderung und gibt eine neue Abfrage an. In diesem Fall wird der Suchordner automatisch neu erstellt. Der RebuildResults-Knoten muss nicht eingeschlossen werden. – Sendet eine Suchanforderung, die den RebuildResults-Knoten enthält. In diesem Fall wird der Server gezwungen, den Suchordner neu zu erstellen. Wenn ein neues Element hinzugefügt wird, erscheint das Element nicht in der Ergebnismenge, bis die Ergebnismenge aktualisiert wird. Wenn ein Element gelöscht wird, filtert der Server das gelöschte Element aus der Ergebnismenge heraus. Der Client SOLLTE alle paar Tage eine neue Suchanforderung mit der angegebenen Abfrage senden und die Option RebuildResults einschließen, um genaue Ergebnisse für diese Abfrage sicherzustellen.
