## PictureRequest class

Gibt an, dass der Client anfordert, dass Fotos in der Serverantwort zurückgegeben werden. Das Bild wird nicht unterstützt, wenn die Protokollversion 12.1 oder 14.0 ist. Enthält die Daten zur Fotoanforderung.

| Name | Beschreibung |

| [MaxPictures](../../aspose.email.clients.activesync.transportlayer/picturerequest/maxpictures) { get; set; } | Begrenzt die Anzahl der Kontaktfotos, die in der Serverantwort zurückgegeben werden. MaxPictures wird nicht unterstützt, wenn die Protokollversion 12.1 oder 14.0 ist. Der Server gibt die ersten N Ergebnisse mit Kontaktfotos zurück, wobei N der Wert von MaxPictures ist. Nachdem das MaxPictures-Limit erreicht ist, gibt der Server den Statuswert 173 (NoPicture) zurück, wenn der Kontakt kein Foto hat, den Statuswert 174 (PictureTooLarge), wenn das Foto des Kontakts zu groß ist, oder den Statuswert 175 (PictureLimitReached), wenn der Kontakt ein hat Foto, aber das MaxPictures-Limit wurde erreicht. Beachten Sie, dass MaxPictures die Anzahl der pro Abfrage zurückgegebenen Kontaktfotos identifiziert. Wenn der Client also drei aufzulösende Empfänger einbezieht und den MaxPictures-Wert in der ResolveRecipients-Befehlsanforderung auf 3 setzt, können maximal 9 Kontaktfotos zurückgegeben werden. |
| [MaxSize](../../aspose.email.clients.activesync.transportlayer/picturerequest/maxsize) { get; set; } | Gibt die maximale Größe eines einzelnen Kontaktfotos an, das in der Antwort zurückgegeben wird, in Bytes. MaxSize wird nicht unterstützt, wenn die Protokollversion 12.1 oder 14.0 ist. Der Maximalwert von MaxSize beträgt 100 KB oder 102400 Bytes. |
