## SyncRequest.HeartbeatInterval property

Gibt die Anzahl von Sekunden an, die der Server eine Antwort verzögern SOLLTE, wenn keine neuen Elemente zu den enthaltenen Ordnern hinzugefügt werden. Gültige Werte für HeartbeatInterval sind 60 bis 3540 Sekunden (59 Minuten). Wenn der Client ein Intervall anfordert, das außerhalb des akzeptablen Bereichs liegt, sendet der Server eine Antwort, die einen Statuswert von 14 und ein Limit-Element enthält. Entweder das HeartbeatInterval oder das Wait können in einer Sync-Befehlsanforderung enthalten sein, aber nicht beide. Wenn beide Elemente enthalten sind, enthält die Serverantwort einen Statuswert von 4.
