## SyncRequest.Wait property

Gibt die Anzahl von Minuten an, die der Server eine Antwort verzögern SOLLTE, wenn keine neuen Elemente zu den enthaltenen Ordnern hinzugefügt werden. Gültige Werte für Wait sind 1 bis 59. Wenn der Client ein Warteintervall anfordert, das außerhalb des zulässigen Bereichs liegt, Der Server sendet eine Antwort, die einen Statuswert von 14 und ein Limit enthält. Entweder Wait oder HeartbeatInterval kann in einer Sync-Befehlsanforderung enthalten sein, aber nicht beides. Wenn beide enthalten sind, enthält die Serverantwort einen Statuswert von 4.
