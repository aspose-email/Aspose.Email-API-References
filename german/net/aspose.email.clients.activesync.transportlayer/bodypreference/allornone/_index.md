## BodyPreference.AllOrNone property

Ein Client kann mehrere BodyPreference-Elemente in eine Befehlsanforderung mit unterschiedlichen Werten für das Type-Element aufnehmen (Abschnitt 2.2.2.22.4). Standardmäßig gibt der Server die Daten gekürzt auf die von TruncationSize angeforderte Größe für das Type-Element zurück, das dem nativen Speicherformat des Body-Elements des Elements entspricht. Wenn der Client jedoch auch das AllOrNone-Element mit dem Wert TRUE zusammen mit dem TruncationSize-Element enthält, weist er den Server an, keine abgeschnittene Antwort für diesen Typ zurückzugeben, wenn die Größe (in Byte) der verfügbaren Daten den Wert überschreitet des TruncationSize-Elements. Beispielsweise kann ein Client diese beiden Elemente verwenden, um anzugeben, dass er partielle RTF-Daten (Rich Text Format) nicht verarbeiten kann (ein Type-Elementwert von 3). Wenn der Client mehrere BodyPreference-Elemente angegeben hat, wählt der Server in diesem Fall das nächste BodyPreference-Element aus, das die maximale Menge an Textkörper an den Client zurückgibt. Angenommen, der Client gibt zwei BodyPreference-Elemente an. Wenn das AllOrNone-Element nicht definiert ist, wird der abgeschnittene oder nicht abgeschnittene Inhalt gesucht, synchronisiert oder abgerufen, als wäre der Wert auf FALSE gesetzt.
