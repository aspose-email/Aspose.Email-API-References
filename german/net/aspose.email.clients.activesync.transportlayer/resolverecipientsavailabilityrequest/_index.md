## ResolveRecipientsAvailabilityRequest class

Zeigt dem Server an, dass Frei/Gebucht-Daten vom Client angefordert werden, und identifiziert die Start- und Endzeit der abzurufenden Frei/Gebucht-Daten. Die Verfügbarkeit wird nicht unterstützt, wenn die Protokollversion 12.1 ist.

| [EndTime](../../aspose.email.clients.activesync.transportlayer/resolverecipientsavailabilityrequest/endtime) { get; set; } | Identifiziert das Ende der vom Client angeforderten Frei/Gebucht-Zeitspanne. Die EndTime wird nicht unterstützt, wenn die Protokollversion 12.1 ist. Wenn der Client einen ungültigen EndTime-Wert sendet, gibt der Server einen Statuswert von 5 Zoll zurück die ResolveRecipients-Befehlsantwort. Wenn keine EndTime in einer Verfügbarkeitsanforderung enthalten ist, verwendet der Server einen Standardendzeitwert von sieben Tagen nach dem StartTime-Wert. Wenn der in der Anforderung angegebene EndTime-Wert kleiner als der StartTime-Wert plus 30 Minuten ist oder die Dauer zwischen dem StartTime-Wert und dem EndTime-Wert größer als eine bestimmte Anzahl von Tagen ist, gibt der Server einen Statuswert von 5 in zurück die ResolveRecipients-Befehlsantwort. Exchange 2010 und Exchange 2013 verwenden 42 Tage; Exchange 2007 SP1 verwendet 62 Tage. |
| [StartTime](../../aspose.email.clients.activesync.transportlayer/resolverecipientsavailabilityrequest/starttime) { get; set; } | Identifiziert den Beginn der vom Client angeforderten Frei-/Gebucht-Zeitspanne. Die Startzeit wird nicht unterstützt, wenn die Protokollversion 12.1 ist. Wenn die Verfügbarkeit in der Anfrage enthalten ist, MUSS die Anfrage auch die Start- und Endzeit enthalten. Wenn der Client einen ungültigen StartTime-Wert sendet, gibt der Server einen Statuswert von 5 in der ResolveRecipients-Befehlsantwort zurück. |
