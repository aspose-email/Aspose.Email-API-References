## PictureRequest class

Indica que el cliente está solicitando que se devuelvan fotos en la respuesta del servidor. La imagen no es compatible cuando la versión del protocolo es 12.1 o 14.0. Contiene los datos relacionados con la solicitud de fotos.

| Nombre | Descripción |
| --- | --- |
| [MaxPictures](../../aspose.email.clients.activesync.transportlayer/picturerequest/maxpictures) { get; set; } | Limita la cantidad de fotos de contacto devueltas en la respuesta del servidor. MaxPictures no es compatible cuando la versión del protocolo es 12.1 o 14.0. El servidor devuelve los primeros N resultados que tienen fotos de contacto, donde N es el valor de MaxPictures. Después de alcanzar el límite MaxPictures, el servidor devuelve el valor de estado 173 (NoPicture) si el contacto no tiene foto, el valor de estado 174 (PictureTooLarge) si la foto del contacto es demasiado grande o el valor de estado 175 (PictureLimitReached) si el contacto tiene una foto. pero se alcanzó el límite de MaxPictures. Tenga en cuenta que MaxPictures identifica el número de fotos de contacto devueltas por consulta. Por lo tanto, si el cliente incluye tres destinatarios para resolver y establece el valor de MaxPictures en 3 en la solicitud de comando ResolveRecipients, se pueden devolver un máximo de 9 fotos de contacto. |
| [MaxSize](../../aspose.email.clients.activesync.transportlayer/picturerequest/maxsize) { get; set; } | Especifica el tamaño máximo de una foto de contacto individual que se devuelve en la respuesta, en bytes. MaxSize no se admite cuando la versión del protocolo es 12.1 o 14.0. El valor máximo de MaxSize es 100 KB o 102400 bytes. |
