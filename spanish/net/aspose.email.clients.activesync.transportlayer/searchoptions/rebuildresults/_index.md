## SearchOptions.RebuildResults property

Obliga al servidor a reconstruir la carpeta de búsqueda (2) que corresponde a una consulta dada. Los resultados de la búsqueda (es decir, el conjunto de resultados) se almacenan en una carpeta de búsqueda en el servidor. De esta forma, cuando un cliente regresa con la misma consulta pero con un nuevo rango de filas, las filas se extraen del conjunto de resultados que está actualmente almacenado en la carpeta de búsqueda. No es necesario reconstruir todo el conjunto de resultados. La carpeta de búsqueda permanece sin cambios hasta que el cliente realiza una de las siguientes acciones para actualizar el conjunto de resultados: : envía una solicitud de búsqueda, especificando una nueva consulta. En este caso, la carpeta de búsqueda se reconstruye automáticamente. No es necesario incluir el nodo RebuildResults. : envía una solicitud de búsqueda que incluye el nodo RebuildResults. En este caso, el servidor se ve obligado a reconstruir la carpeta de búsqueda. Si se agrega un nuevo elemento, el elemento no aparece en el conjunto de resultados hasta que se actualiza el conjunto de resultados. Si se elimina un elemento, el servidor filtrará el elemento eliminado del conjunto de resultados. El cliente DEBERÍA enviar una nueva solicitud de búsqueda con la consulta dada e incluir la opción RebuildResults cada pocos días para garantizar resultados precisos para esa consulta.
