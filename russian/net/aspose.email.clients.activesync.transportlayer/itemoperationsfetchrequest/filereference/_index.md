## ItemOperationsFetchRequest.FileReference property

Задает уникальный идентификатор, назначаемый сервером каждому вложению. В запросе ItemOperations может существовать только один идентификатор FileReference для каждой выборки. Нарушение этого ограничения приводит к тому, что значение элемента Status, равное 2, возвращается с сервера. Однако клиент может получить несколько вложений, используя один узел Fetch для каждого вложения.

```csharp
public string FileReference { get; set; }
```
