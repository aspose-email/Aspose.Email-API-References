## ResolveRecipientsAvailabilityRequest class

Указывает серверу, что данные о занятости запрашиваются клиентом, и определяет время начала и окончания получения данных о занятости. Доступность не поддерживается, если используется версия протокола 12.1.

| [EndTime](../../aspose.email.clients.activesync.transportlayer/resolverecipientsavailabilityrequest/endtime) { get; set; } | Идентифицирует конец диапазона времени занятости, запрошенного клиентом. EndTime не поддерживается при версии протокола 12.1. Если клиент отправляет недопустимое значение EndTime, то сервер возвращает значение состояния 5 в ответ команды ResolveRecipients. Если EndTime не включен в запрос доступности, сервер использует значение времени окончания по умолчанию, равное семи дням после значения StartTime. Если значение EndTime, указанное в запросе, меньше, чем значение StartTime плюс 30 минут, или продолжительность, охватываемая значением StartTime и значением EndTime, больше определенного количества дней, то сервер возвращает значение состояния 5 в ответ команды ResolveRecipients. Exchange 2010 и Exchange 2013 используют 42 дня; Exchange 2007 SP1 использует 62 дня. |
