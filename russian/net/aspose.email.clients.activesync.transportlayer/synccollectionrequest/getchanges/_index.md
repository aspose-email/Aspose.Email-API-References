## SyncCollectionRequest.GetChanges property

Запрашивает, чтобы сервер включил в свой ответ любые ожидающие изменения в коллекции, указанной элементом ServerId (раздел 2.2.3.151.6). Если с момента последней синхронизации произошли изменения, ответ сервера включает элемент Commands (раздел 2.2.3.32) который содержит добавления, удаления и изменения. Если клиент не хочет возвращать изменения сервера, запрос ДОЛЖЕН включать элемент GetChanges со значением FALSE. Значение TRUE указывает, что клиент хочет, чтобы серверные изменения были возвращены. Значение TRUE предполагается, когда элемент GetChanges пуст. Когда в запросе отсутствует элемент GetChanges, поведение зависит от значения элемента SyncKey, как указано в разделе 2.2.3.166.4. Если элемент SyncKey имеет значение 0, то запрос обрабатывается так, как если бы для элемента GetChanges было установлено значение FALSE. Если элемент SyncKey имеет ненулевое значение, то запрос обрабатывается так, как если бы для элемента GetChanges было установлено значение TRUE.

public bool? GetChanges { get; set; }
