## BackupSettings.NumberOfAttemptsToRrepeat property

Получает или задает значение, которое определяет количество попыток повторить неудачную операцию В случае, если какая-то команда IMAP внутри операции резервного копирования возвращает неудачный результат, клиент IMAP пытается повторить эту операцию снова в соответствии с заданным количеством раз. Например, если операция FETCH возвращает ошибку AE_1_1_0243 FETCH 219 (BODY) AE_1_1_0243 NO[UNAVAILABLE] FETCH Служба временно недоступна Клиент пытается выполнить его снова.

public int NumberOfAttemptsToRrepeat { get; set; }
