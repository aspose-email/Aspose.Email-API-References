## ImapSpecialFolderTypes enumeration

Представляет перечисление почтовых ящиков специального назначения Подробнее см. в RFC6154 http://tools.ietf.org /html/rfc6154

```csharp
public enum ImapSpecialFolderTypes
```

### Ценности

| Имя | Ценность | Описание |
| --- | --- | --- |
| NotSpecified | `0` | Если поддерживается расширение IMAP LIST для специальных почтовых ящиков (RFC6154), это означает, что почтовый ящик является обычным и не используется для специальных целей. |
| All | `1` | В этом почтовом ящике представлены все сообщения в хранилище сообщений пользователя. Реализации МОГУТ пропускать некоторые сообщения, например, в \Trash и \Junk. Когда это специальное использование поддерживается, оно почти наверняка представляет собой виртуальный почтовый ящик. |
| Archive | `2` | Этот почтовый ящик используется для архивирования сообщений. Значение "архивного" почтового ящика зависит от сервера; как правило, он будет использоваться для извлечения сообщений из папки «Входящие» или иным образом держать их подальше от пользователя, но при этом делать их доступными. |
| Drafts | `3` | Этот почтовый ящик обычно используется для хранения черновиков сообщений, которые составляются, но еще не были отправлены. В некоторых реализациях сервера это может быть виртуальный почтовый ящик, содержащий сообщения из других почтовых ящиков, помеченных флагом сообщений "\Draft". В качестве альтернативы, это может быть просто совет, чтобы клиент помещал сюда черновики. |
| Flagged | `4` | В этом почтовом ящике представлены все сообщения, помеченные тем или иным образом как "важные". Когда это специальное использование поддерживается, оно, вероятно, представляет собой виртуальный почтовый ящик, собирающий сообщения (из других почтовых ящиков), отмеченные флагом сообщения "\Flagged". |
| Junk | `5` | В этом почтовом ящике хранятся сообщения, которые считаются нежелательной почтой. Некоторые реализации сервера могут помещать сюда сообщения автоматически. В качестве альтернативы, это может быть просто совет спам-фильтру на стороне клиента. |
| Sent | `6` | Этот почтовый ящик используется для хранения копий отправленных сообщений. Некоторые реализации сервера могут помещать сюда сообщения автоматически. В качестве альтернативы, это может быть просто совет, чтобы клиент сохранял здесь отправленные сообщения. |
| Trash | `7` | Этот почтовый ящик используется для хранения сообщений, которые были удалены или отмечены для удаления. В некоторых реализациях сервера это может быть виртуальный почтовый ящик, содержащий сообщения из других почтовых ящиков, помеченных флагом сообщений "\Deleted". В качестве альтернативы, это может быть просто совет, что клиент, решивший не использовать модель IMAP "\Deleted", должен использовать ее в качестве места для корзины. В реализациях серверов, которые строго ожидают модель IMAP "\Deleted", это специальное использование, скорее всего, не будет поддерживаться. |
| Important | `8` | Этот почтовый ящик используется для хранения сообщений, помеченных как важные. Атрибут почтового ящика "\Important" указывает на то, что почтовый ящик содержит сообщения, которые, вероятно, важны для пользователя. В реализации, которая также поддерживает ключевое слово "$Important", это специальное использование, вероятно, будет представлять виртуальный почтовый ящик, собирающий сообщения (из других почтовых ящиков), помеченные ключевым словом "$Important". В других реализациях система может автоматически помещать туда сообщения на основе тех же видов эвристики, которые отмечены для ключевого слова "$Important". Различия между "\Important" и "\Flagged" для почтовых ящиков аналогичны различиям между "$Important" и "\Flagged" для сообщений. https://tools.ietf.org/html/rfc8457 |
