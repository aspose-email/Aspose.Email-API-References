## SyncCollectionRequest.GetChanges property

请求服务器在其响应中包括对由 ServerId 元素指定的集合的任何未决更改（第 2.2.3.151.6 节）。 如果自上次同步以来发生了更改，则服务器响应包括一个 Commands 元素（第 2.2.3.32 节） 包含添加、删除和更改。 如果客户端不希望返回服务器更改，则请求必须包含值为 FALSE 的 GetChanges 元素。 TRUE 值表示客户端希望返回服务器更改。当 GetChanges 元素为空时，假定值为 TRUE。 当请求中不存在 GetChanges 元素时，行为取决于 SyncKey 元素的值，如第 2.2.3.166.4 节中所述。 如果 SyncKey 元素的值为 0，则处理请求时就像 GetChanges 元素设置为 FALSE。 如果 SyncKey 元素具有非零值，则处理请求，就像 GetChanges 元素设置为 TRUE 一样。

```csharp
public bool? GetChanges { get; set; }
```
