## SearchResult class

单个匹配邮箱项目的容器。 当正在搜索的商店是邮箱时: - 在邮箱中找到的每个匹配项都有一个 Result 元素。 如果未找到匹配项，则响应 XML 的 Store 容器元素中存在一个空的 Result 元素。 - 在 Result 元素内，Properties 元素包含邮箱项目的请求属性列表。 当正在搜索的商店是文档库时: - 搜索响应中返回的第一个结果是 LinkId 指向的根文件夹或项目的元数据价值是指向。 如果不需要，客户端可以选择忽略此条目。 - 如果请求中的 documentlibrary:LinkId 元素值指向一个文件夹，则该文件夹的元数据属性作为第一项返回，该文件夹的内容作为后续结果返回。 Range 适用于这些结果，没有区别；例如，索引 0 将始终用于链接指向的根项。 - 如果请求中的 documentlibrary:LinkId 元素值指向一个项目，则只返回一个结果:项目的元数据。 - 在 Result 元素内，Properties 元素包含邮箱项目的请求属性列表。
