## DSNAction enumeration

指示报告-MTA 作为其尝试传递消息的结果而执行的操作。

```csharp
public enum DSNAction
```

### 价值观

| 姓名 | 价值 | 描述 |
| --- | --- | --- |
| None | `0` | 无动作。 |
| Delayed | `1` | 表示报告 MTA 迄今为止无法传递或中继邮件，但它将继续尝试这样做。当消息被进一步延迟或成功传递，或者如果传递尝试后来被放弃时，可能会发出额外的通知消息。 |
| Delivered | `2` | 表示邮件已成功投递到发件人指定的收件人地址，其中包括“投递”到邮件列表扩展器。它并不表示该消息已被阅读。这是一个终端状态，不应期望该收件人有进一步的 DSN。 |
| Expanded | `3` | 表示消息已成功传递到发件人指定的收件人地址，并由 Reporting-MTA 转发到该目的地之外的多个其他收件人地址。 “expanded”的动作值与 Delivered 的不同之处在于“expanded”不是最终状态。可以提供进一步的“失败”和/或“延迟”通知。 |
| Failed | `4` | 表示无法将邮件传递给收件人。报告 MTA 已放弃将邮件传递给该收件人的任何尝试。预计不会有进一步的通知。 |
| Relayed | `5` | 表示消息已被中继或网关传送到不承担在成功传递后生成 DSN 的责任的环境中。除非发件人已请求此收件人成功递送的通知，否则不应使用此操作值。 |
