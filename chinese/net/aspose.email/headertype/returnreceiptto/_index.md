## HeaderType.ReturnReceiptTo property

发送者可以通过包含这个头域来请求回执。 回执被发送到电子邮件的 Return-Path 地址，而不是 Return-Receipt-To 标头字段中指定的地址。 此标头是非标准的，并且大多不受支持。 使用 Disposition-Notification-To 代替。 即使支持，也不能保证发送收据。

public static HeaderType ReturnReceiptTo { get; }
